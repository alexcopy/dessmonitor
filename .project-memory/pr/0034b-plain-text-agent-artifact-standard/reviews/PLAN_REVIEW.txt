MIXED-FORMAT REVIEW

Mixed-format behavior is defined:

  - For allowlisted non-numeric directories: if a canonical artifact and
    a legacy artifact for the same stage both exist, the validator reports
    an ambiguity error.

  - For canonical directories: legacy names are unconditionally rejected,
    so the mixed state cannot arise (the legacy file is rejected first).

  - For legacy directories: no format checks are applied, so mixed
    canonical+legacy coexistence is accepted without error.

The plan does not define mixed-format behavior for legacy (pre-boundary)
numeric directories that coincidentally have both PLAN.md and PLAN.txt.
This is an extremely unlikely edge case.  The legacy stage detection
would see PLAN.md and classify it as DRAFT; the presence of PLAN.txt
would be ignored because no format checks are applied.  The coder could
add an explicit informational warning if desired.  Not a blocker.
