FINAL SUMMARY

The implementation addresses both production regressions. DevicePropertyMapping
provides a typed, immutable mapping resolved once during initialization and
consumed consistently by both command and observation paths. The universal
switch_1 fallback is removed; devices without explicit mapping receive
mapping_validity=invalid and safe_error. Five competing command methods are
consolidated into one canonical internal path with three public methods
(switch_on, switch_off, set_numeric). Command acceptance never updates
canonical observation.

The StartupResetCoordinator commands every available valid binary switch OFF
at startup with 500ms inter-command delay, performs immediate one-shot
observation via refresh_once(), and gates autonomous automation behind fresh
confirmed OFF for every target. Timeout never opens the gate. Life-support
switches are not exempt. Sensors receive no commands. The reset runs once
per process; a full restart re-executes it. The web host starts early so
operators can see reset progress.

Read model and dashboard expose startup_reset_status, mapping_status, and
per-device reset results. No raw property codes, IDs, keys, or command values
are exposed. UNKNOWN remains currently_on=null. The dashboard remains
read-only.

All 42 deterministic validation tests pass with fake devices, injectable
clocks, and no real network or Tuya access. The implementation is ready for
commit and Pull Request.
