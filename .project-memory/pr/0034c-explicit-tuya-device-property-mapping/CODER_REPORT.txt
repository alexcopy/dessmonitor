DEVICE PROPERTY MAPPING
DevicePropertyMapping (frozen dataclass):
  control_property: str | None
  state_property: str | None
  command_kind: CommandKind (BINARY, NUMERIC, NONE)
  startup_off_value: bool | int | None
  command_capable: bool (derived)
  observable: bool (derived)
  mapping_validity: MappingValidity (VALID, INVALID, UNAVAILABLE)
  mapping_source: MappingSource (EXPLICIT_CONFIG, EXPLICIT_CONTROL_KEY,
    EXPLICIT_STATE_KEY, MULTI_SWITCH_KEY, LEGACY_CHANNEL, LEGACY_API_SW,
    PUMP_COMPAT, INFERRED, FALLBACK_REJECTED)
  safe_error: str | None

MAPPING PRECEDENCE
Multi-switch child:
  control_property = sw_key (YAML dict key like "switch_1", "switch_3")
  state_property = sw_key (same as control)
  command_kind = BINARY, startup_off_value = false
  mapping_source = MULTI_SWITCH_KEY

Single switch:
  control_property = control_key or channel or api_sw
  NO universal switch_1 fallback
  state_property = state_key or control_property
  If all absent: mapping_validity=INVALID, safe_error="no-control-property"

Pump:
  control_property = control_key or channel or api_sw or "P"
  state_property = state_key or p_code or "Power"
  Bounded compat rule with mapping_source=PUMP_COMPAT when defaults used

Sensor (ANALOG_TYPES):
  control_property = None (explicitly)
  command_kind = NONE
  state_property = state_key or channel or api_sw or None

Unknown device_type:
  control_property = control_key or None
  command_kind = BINARY (conservative)
  If no properties: mapping_validity=INVALID

STARTUP RESET COORDINATOR
Class: StartupResetCoordinator
Timeout: 120.0 seconds
Retry interval: 30.0 seconds
Inter-command delay: 0.5 seconds

STARTUP TARGET SELECTION
Included: available=true, mapping_validity=valid, command_kind=binary,
  control_property non-empty, state_property non-empty
Included: multi-switch children, life-support binary switches
Excluded: sensors (command_kind=none), numeric without explicit startup_off_value,
  invalid mappings, unavailable devices

STARTUP RESET SEQUENCE
1. Build target set
2. Submit OFF commands with 500ms inter-command delay
3. Immediate observation refresh via refresh_once()
4. Bounded retries (30s interval, 120s timeout)
5. All confirmed OFF -> gate opens
6. Timeout -> gate stays closed, degraded mode
7. Later fresh OFF observations can open gate after timeout

STARTUP RESET GATE
Flag: is_gate_open (initially False)
Set to True only when every required target has fresh confirmed OFF
While closed: SmartHomeController._switch_loop skips all switch decisions
Pump loop NOT blocked by gate
Gate stays open once opened; full process restart resets it

AUTOMATION RELEASE
SmartHomeController receives startup_reset_coordinator parameter
_switch_loop checks is_gate_open at top of each iteration
Gate check before reading battery voltage or making switch decisions
Timeout never opens the gate
