   - Public function: evaluate_readiness(readiness_input) -> ReadinessResult
   - Evaluates 8 blocking conditions in deterministic order:
     a) Empty/invalid load_id → "invalid-load-id", NO_ACTION
     b) Invalid/missing voltage → "invalid-voltage", NO_ACTION
     c) Voltage below minimum_voltage_to_switch_on → "below-switch-on-voltage", PREFER_OFF
     d) Grid/mains/network power with discretionary load → "grid-or-mains-conservation", PREFER_OFF
     e) Outside allowed_time_windows → "outside-allowed-time-window", PREFER_OFF
     f) Cooldown period active → "cooldown-active", NO_ACTION
     g) Weather skip (cloudy/rainy/storm/snowy) → "weather-skip", PREFER_OFF
     h) Evening reserve protection → "evening-reserve-protected", FORCE_OFF
   - All 9 required reason strings implemented: ready, invalid-load-id, invalid-voltage,
     below-switch-on-voltage, grid-or-mains-conservation, outside-allowed-time-window,
     cooldown-active, weather-skip, evening-reserve-protected.
   - Pure function: no side effects, deterministic, uses context.voltage.timestamp
