DEFECTS FIXED

1. capture_signals was @asynccontextmanager + async def.
   uvicorn 0.51.0 Server.serve() uses ``with self.capture_signals():``
   (synchronous context manager). Changed to @contextmanager + def.

2. _wait_for_startup did not monitor server_task. If the task died
   before server.started became true, the code waited the full 5 s
   timeout then reported only "web-host-startup-timeout". Fixed:
   _wait_for_startup now accepts server_task, checks task.done()
   on each poll iteration, and re-raises the original exception.

3. Startup failure traces were only logged via the optional logger
   (typically important_log → logs/important.log). Added
   _log_startup_failure() that also prints the complete traceback
   to sys.stderr so container stdout captures it.

4. Dead _SignalSafeServer module-level class removed (was shadowed
   by local class inside _make_signal_safe_server).

5. Integration test had ``|| echo SKIP`` fallback — removed.
   Test is now mandatory; any startup exception exits non-zero.
