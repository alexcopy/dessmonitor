OBSERVATION CONTRACT REVIEW

DeviceObservationState (frozen dataclass) in app/devices/device_observation.py:
  observed_state: ObservationValue (ON="on", OFF="off", UNKNOWN="unknown")
  observed_at: datetime | None (UTC, timezone-aware)
  observation_source: str | None ("tuya" or None)
  Properties: is_on, is_off, is_unknown, has_observation

ObservationFreshness enum: FRESH="fresh", STALE="stale", UNAVAILABLE="unavailable"
compute_freshness() function computes dynamically from observation and thresholds.

No raw-dictionary state as canonical contract. Contract is explicit, typed, immutable.
