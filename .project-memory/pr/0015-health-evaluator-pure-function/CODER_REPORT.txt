BEHAVIOR IMPLEMENTED
====================
1. Created app/control/health.py — Pure, deterministic health evaluator.
   - Public function: evaluate_health(health_input) -> HealthCheckResult
   - Evaluates 7 health conditions in deterministic order:
     a) Empty/invalid load_id → UNKNOWN, "invalid-load-id", "inspect-device"
     b) Unreachable (expected present, observed missing) → UNREACHABLE, "unreachable", "refresh-status"
     c) Stale status (>300s) → STALE, "stale-status", "refresh-status"
     d) Repeated failures (>=3) → UNREACHABLE, "repeated-failures", "investigate-failures"
     e) State mismatch → MISMATCH, "state-mismatch", "inspect-device"
     f) Healthy (both present, equal, no stale/failure) → HEALTHY, "healthy", "none"
     g) Unknown (insufficient data) → UNKNOWN, "unknown-state", "refresh-status"
   - All 7 required reason strings implemented.
   - Module-level constants: DEFAULT_STALE_STATUS_SECONDS=300.0,
     DEFAULT_FAILURE_COUNT_THRESHOLD=3.
   - Pure function: no side effects, deterministic, no file/network/env-var reads,
     no hardware calls, no switch retry, no system clock calls.
   - Uses only Python stdlib + app.control.energy_policy types.
